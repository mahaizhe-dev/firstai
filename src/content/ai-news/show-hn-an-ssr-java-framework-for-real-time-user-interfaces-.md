Why?- Implement stateful, server-driven rendering (SSR over WebSockets) framework for Java simlilar to Phoenix LiveView.- Introduce a user interfaces component system that enables easy composition.- Provide a vertically integrated solution targeting near-zero runtime dependencies. It takes an "all-included" monolithic approach, inspired by the Linux kernel philosophy.- Embrace AI agent support for development, distribution, and runtime. Paradoxically, at the same time, it empowers developers to fully own/understand the stack: a small core (~12K LOC) written in modern Java with no annotations and no implicit execution flows.
