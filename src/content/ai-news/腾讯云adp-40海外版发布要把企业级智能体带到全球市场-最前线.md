腾讯云的企业级智能体平台，正式出海了。 7月18日，在2026世界人工智能大会上，腾讯云正式发布了智能体开发平台 ADP 4.0海外版，同步升级智能工作台、Claw 模式、Skill 广场三大核心模块，围绕触达、交互、生态、连接四大能力做了全面国际化适配。 ADP 的全称是 Agent Development Platform，定位为企业级 AgentOps 平台，覆盖智能体的构建、分发和治理全生命周期。 简单来说，它解决的是企业智能体从建立、运行到管理的问题。 智能体开发平台 ADP 4.0海外版 这次企业级智能体海外版的核心升级集中在四个方面：一是渠道，Claw 应用支持 LINE、Telegram 等主流海外社交渠道，一站发布即可跨区域触达全球用户； 而在交互体验上，智能工作台支持自定义时区与多区域调度，模型输出可跟随用户输入语言自动响应； 在开放生态上，模型广场已经接入多家国际主流大模型，平台 Skill 完成国际化适配；并且，平台深度接入 Google Workspace、Confluence、Jira 等海外主流 SaaS 应用。 ADP 4.0的一个关键能力升级是 Agent 与 Workflow 双向互调。吴运声在采访中解释了这个设计的逻辑：企业在实际使用中，有些事情是模糊的、开放的，适合用 Agent 方式处理，这会消耗较多 Token；但一些确定性的流程，可以用 Workflow 来做，不需要反复消耗 Token。两者结合，既满足业务需求，又降低成本。 这套逻辑在国内也已经有了标杆案例。比如，找钢网基于 ADP 4.0 Claw 模式搭建了企业中台数字员工，接入企业微信，覆盖订单、物流、财务、仓库、发票等多类场景。Agent 负责理解开放问题，Workflow 负责执行严谨业务流程，两者在同一个应用中协同工作。 海外市场，ADP 4.0同样已有落地成果。如今，某大型医疗集团利用 ADP 构建了覆盖智能导诊、门诊预约、急诊风险识别等全流程的 AI 医疗助手，预计可独立处理80%以上常规咨询和预约，整体预约效率提升50%以上，患者平均等待时间缩短30%。而另一家海外头部运营商基于 ADP 重构统一客服平台后，整体运维成本预计降低30%-40%，新业务上线周期从数周缩短至数天，应用开发效率预计提升70%以上。 吴运声在采访中表示，ADP 的客户数较去年至少翻倍增长。他将增长归因于两点：一是今年 Agent Loop 能力变强，以前不太好做的事情在能力上已经可以支持；二是客户在日常生活和工作中已经被教育了，教育成本大大降低。 尽管当前ADP仍以国内市场为主，海外重点看东南亚等区域，会综合考虑市场人口、基建情况、政治环境和对华关系。在印尼，腾讯云已经做了不少私有化项目，在企业内部安装部署智能体。 值得注意的是，吴运声在发布中还提出了“十大行业百大场景生态计划”，邀请生态伙伴从客服、经营分析、交易自动化等高频场景切入，依托 ADP 底座和伙伴的行业 Know-how，把智能体带进千行百业。同时，腾讯云还升级了腾讯乐享 Agentic 知识库作为知识底座，与 ADP 的能力底座形成配合。 2026年，OpenClaw带来的Agent热潮，也让很多企业迅速将Agent应用到生产流程中。当被问及 WorkBuddy 和 ADP 的关系时，吴运声打了个比方：WorkBuddy 像是员工个人的智能助手，企业管理者未必知道员工用了什么；而 ADP 是云端一体化方案，做好的智能体可以发布出来供团队使用，使用情况对企业完全透明。两者是并存关系，面向不同场景。 进入2026年，企业级智能体赛道正在快速升温。吴运声认为，企业最关心的是可信、可控——“我得知道到底是谁做了什么，后面所有东西都要可审计、是安全的。”这也是 ADP 相较于个人级智能体产品的核心差异：管控性更强，可以设定权限，支持全链路可观测和审计，并能直接集成到企业原有的 CRM 或 OA 系统中。 从更大的视角看，这次 ADP 4.0海外版发布，是腾讯云将国内打磨成熟的企业级智能体能力向全球市场输出的关键一步。中国模型的性价比优势已经被海外中小企业认知，而企业级智能体平台的出海，意味着竞争从 Token 层上升到了应用层。
